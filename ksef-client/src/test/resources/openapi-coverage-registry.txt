# OpenAPI path/operation coverage registry
#
# Each line is either a spec path or a (METHOD, path) operation tuple
# exercised by at least one WireMock contract test. Comments after `#`
# and blank lines are ignored.
#
# OpenApiPathCoverageTest reads `ksef-client/openapi/open-api.json` and
# asserts every path AND every operation tuple appears here. Adding an
# endpoint to the SDK without adding it here (and a corresponding test)
# fails the gate.
#
# Format options (Codex round-9 fresh review M2):
#   /tokens                                  # path-only entry — counts as covering ALL methods on /tokens
#   POST /tokens                             # method-specific tuple — only POST /tokens
#   GET /tokens/{referenceNumber}            # method-specific tuple
#
# Path-only entries are kept for backwards compatibility; new tests that
# exercise only one HTTP method on a multi-method path SHOULD register
# method-specific tuples so spec drift on a sibling method (e.g. a new
# DELETE on a path that only had GET+POST tested) is detected.
#
# Do NOT prepend the /v2 prefix to paths.
